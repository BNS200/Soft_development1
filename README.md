# Soft_development

# Постановка задачи

Написать программу с консольным интерфейсом, которая выполняет слежение за выбранными файлами.
Ограничимся двумя характеристиками за изменениями которых выполняется слежение:
1. Существование файла;
2. Размер файла.
Программа будет выводить на консоль уведомление о произошедших изменениях в файле.

# Предлагаемое решение

## 1. Общая архитектура

Решение базируется на архитектуре, основанной на сигнально-слотовой модели Qt. Применены следующие паттерны проектирования:

1. Singleton – для класса fileManager, обеспечивающий единственный экземпляр менеджера файлов.

## 2. Структура классов

### 2.1. Класс fileManager

**Назначение:** сущность для управления и мониторинга файлов.

**Основные методы:** 
- `Instance()` – статический метод для получения единственного экземпляра
- `addFile()` – добавление в список отслеживаемых файлов
- `removeFile()` – удаление из списка отслеживаемых файлов
- `setLogger()` – установка логгера для обработки событий
- `updateFiles()` – слот для проверки состояния файла с определенной периодичность

**Сигналы:**
- `fileCreated()` – сигнал, вызывающий слот logCreated()
- `fileChanged()` – сигнал, вызывающий слот logChanged()
- `fileDeleted()` – сигнал, вызывающий слот logDeleted()
- `fileExists()` – сигнал, вызывающий слот logExists()
- `fileNotExists()` – сигнал, вызывающий слот logNotExists()

### 2.2. Класс logger

**Назначение:** обработка ввода и вывода информации о событиях файловой системы.

**Основные методы:** 
- `logCreated()` – слот вызывающийся при создании файла 
- `logExists()` – слот вызывающийся при существовании файла
- `logDeleted()` – слот вызывающийся при удалении файла 
- `logNotExists()` – слот вызывающийся при не существовании файла
- `logChanged()` – слот вызывающийся при изменении файла
- `print()` – метод отвечающий за вывод в консоль 
- `formatSize()` – метод отвечающий за форматирование размера при выводе в консоль

## 3. UML диаграмма

## 4. Паттерн Singleton

**Назначение:** гарантирует, что класс имеет только один экземпляр, и предоставляет доступ к нему из любой точки кода.

**Архитектура Singleton в моем коде:**
- Приватный конструктор 
- Статический метод Instance()
- Конструктор копирования и оператор присваивания удалены

# Инструкция пользователя

## 1. Сборка программы

**Требования для сборки:**
- Qt: версия 5 или 6 
- CMake: версия 3.1 или выше
- Компилятор: с поддержкой C++17

**Инструкция по сборке (macOS/Linux):**
- Настройка проекта: `cmake -B build -S .`
- Компиляция проекта: `cmake --build build`
- Запуск программы: `./build/Lab1`

## 2. Настройка программы

Для изменения списка файлов необходимо отредактировать файл main.cpp, переходим в данный файл и ищем строки:
- `manager.addFile("/путь/к/файлу1.txt");`
- `manager.addFile("/путь/к/файлу2.txt");`

Чтобы добавить файл для мониторинга, необходимо прописать строку ниже или выше представленных:
- `manager.addFile("/путь/к/файлу3.txt");`

Чтобы изменить частоту проверки, необходимо в функции main найти строку:
- `timer.start(100);`
и заменить число, стоящее в скобках; значение указывается в миллисекундах.

## 3. Пример работы программы

**Создание файла file1.txt и запись в него строки "Hello":**
- `echo "Hello" > file1.txt`

**Создание пустого файла file2.txt:**
- `touch file2.txt`

**Вывод программы:**
- `"08:33:28.655""Файл'/Users/nikitabufalov/work2/Soft_development/Lab1/file1.txt' существует, раззмер '6B'"`

**Удаление файла file1.txt:** 
- `rm file1.txt`

**Вывод программы:**
- `"08:37:11.747""Файл'/Users/nikitabufalov/work2/Soft_development/Lab1/file1.txt' удален"`
- `"08:37:11.747""Файл'/Users/nikitabufalov/work2/Soft_development/Lab1/file1.txt' не существует"`
